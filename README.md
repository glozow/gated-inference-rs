# gated-inference-rs

Signature-gated LLM inference server. Rust + axum + llama.cpp, built
reproducibly with Nix.

The server pins one ECDSA-secp256k1 public key at startup. Every
`/generate` request must carry a payload signed by that key; anything
else is rejected before llama.cpp is touched. The server signs its own
response with a boot-ephemeral key so archived blobs can be re-verified
later.

See the workspace-level [CLAUDE.md](../CLAUDE.md) for the broader trust
model and how this service fits alongside `guard/` and `gated-inference/`.

## Why Rust + Nix

- **`Cargo.lock` + `flake.lock`**: end-to-end reproducibility. Same
  lockfiles → bit-identical binary and OCI image digest.
- **Same `libsecp256k1` on server and Python client**: nixpkgs'
  `secp256k1` is built once and linked by both the Rust `secp256k1`
  crate and Python's `coincurve` in the devShell. No spec-edge drift.
- **Sandboxed build**: `nix build` runs with no network access. All
  inputs are declared; all outputs are reproducible.

## Layout

```
gated-inference-rs/
├── flake.nix                  # pins nixpkgs → pins every dep
├── flake.lock                 # generated by `nix flake lock`
├── Cargo.toml
├── Cargo.lock                 # generated by `cargo build`; commit after first build
├── rust-toolchain.toml        # pins rustc version
├── src/
│   ├── lib.rs
│   ├── canonical_json.rs      # sort_keys + compact separators, matches Python
│   ├── verifier.rs            # ECDSA verify + LRU replay + timestamp freshness
│   ├── signer.rs              # boot-ephemeral ECDSA signer for responses
│   ├── llama.rs               # llama-cpp-2 wrapper
│   └── main.rs                # axum server: /health /pubkey /generate
├── tests/interop.rs           # end-to-end sign/verify roundtrip
└── python-client/
    ├── sign_request.py        # coincurve-based signer + CLI
    └── README.md
```

## Dev workflow

```bash
# First-time setup — generates Cargo.lock and flake.lock.
cd gated-inference-rs
nix develop               # drops you into a shell with pinned rustc, cmake,
                          # llama.cpp toolchain, python + coincurve, jq
cargo build --release
```

### Run locally

```bash
# 1. Generate a client keypair
python python-client/sign_request.py --gen-key
# privkey_hex: <KEEP SECRET>
# pubkey_hex:  02abcd...

# 2. Start the server with that pubkey authorized
AUTHORIZED_PUBKEY=02abcd... \
  LLM_MODEL_PATH=$PWD/../scaffold/models/main.gguf \
  PORT=9300 \
  cargo run --release

# 3. Sign and send a request
python python-client/sign_request.py \
  --privkey-hex <privkey_hex> \
  --prompt "What is the capital of France?" \
  --url http://localhost:9300/generate
```

### Endpoints

| Endpoint    | Method | Description |
|-------------|--------|-------------|
| `/health`   | GET    | status + boot_time + pinned authorized_pubkey |
| `/pubkey`   | GET    | server's response-signing pubkey (boot-ephemeral) |
| `/generate` | POST   | body = `{payload, signature}`; runs LLM, returns signed result |

### Env vars

| Var                 | Default              | Notes |
|---------------------|----------------------|-------|
| `AUTHORIZED_PUBKEY` | — (required)         | 33-byte SEC1 compressed hex |
| `LLM_MODEL_PATH`    | `/models/main.gguf`  | GGUF file |
| `LLM_CTX`           | `4096`               | context tokens |
| `LLM_MAX_TOKENS`    | `256`                | max generated tokens |
| `MAX_AGE_SECS`      | `60`                 | timestamp freshness window |
| `NONCE_CACHE_SIZE`  | `10000`              | LRU size for replay protection |
| `PORT`              | `8080`               | TCP port |
| `RUST_LOG`          | unset                | tracing filter |
